Идеальный компонент query builder является построителем универсальных запросов к базе.
Цель его - сократить повторение написание Sql запросов для запроса или изменения данных в разных таблицах.
Так же компонент позволяет передавать уточняющие/ограничивающие параметры.

Компонент имеет 4 основных действия с базой - выборка, вставка, правка и удаление записи.
Так же компонент позволяет избегать множественных подключений к базе и работает только с одним подключением в контексте
исполнения одного запроса к серверу во время жизни скрипта.

Метод select всегда должен возвращать данные или null при их отсутствии в таблице.
Метод insert возвращает номер только что созданной записи для дальнейшей работы с ней в случае необходимости.
Методы update, delete требуют еще один параметр - идентификатор записи в таблице, с которым будут произведены действия.

Querybuilder использует современный, встроенный в php ООП подход - PDO - для работы с разными базами данных.
Основным моментом PDO является усиленная защита запроса и фильтрация параметров при помощи placeholder.
Основной сложностью компонента Querybuilder является обработка массива с параметрами, построение строки sql с placrholder
и привязка параметров к ним.
